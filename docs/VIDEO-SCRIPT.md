# OpenDeepSeek 视频文案

## 标题候选

1. DeepSeek 不该只是聊天框：我把它接成了一个真 Agent
2. 便宜的 DeepSeek V4，不拿来自动化电脑太可惜了
3. 用 Open WebUI + Hermes，把 DeepSeek 变成手机里的个人助理

## 3 分钟口播脚本

### 0:00-0:15 开头钩子

DeepSeek 官方聊天你们用过吧？  
能聊，能搜，但说实话，它还是一个聊天框。

真正可惜的是：DeepSeek V4 的 Agent 能力其实很强，价格又便宜，但大多数人只是拿它问答、写两段文字。  
这就像你买了一台挖掘机，最后只拿它铲了一勺沙。

### 0:15-0:45 痛点

普通聊天工具最大的问题是：它不真的做事。

你让它记住偏好，它下次可能忘。  
你让它提醒你，它可能只是嘴上答应。  
你让它看看电脑文件，它根本没有权限。  
你想让它做网页、写周报、生成 PPT、定时跑任务，它最后还是给你一段建议。

所以我做 OpenDeepSeek 的初衷不是再造一个聊天 UI。  
我要的是：让 DeepSeek API 真正跑在 Agent 上。

### 0:45-1:25 解决方案

OpenDeepSeek 是三层：

第一层，Open WebUI，负责好用的网页、手机 PWA、桌面体验。  
第二层，Hermes Agent，负责记忆、技能、文件、终端、定时任务、子代理。  
第三层，DeepSeek V4，负责便宜又强的推理能力。

这样一来，你在手机上说一句话，不是只得到一段回答，而是能触发一个真实动作。

比如：  
“看看我的桌面有什么，帮我整理一下。”  
它会真的访问 `/host/Desktop`，总结文件类型，生成整理方案。  

“帮我做一个个人主页。”  
它会真的在电脑里写出 `index.html`。  

“10 分钟后提醒我回来检查。”  
它会真的创建 Hermes Cron 任务。

### 1:25-2:10 屏幕演示节奏

镜头一：打开 OpenDeepSeek，首屏不是英文模板，而是中文 Agent 模板。  
点“整理我的桌面”。  
展示它使用 `/host/Desktop`，但不移动文件，只生成整理方案。

镜头二：点“生成一个网页”。  
展示它创建 `/host/OpenDeepSeek-Outputs/site/index.html`，然后打开网页。  
强调：这不是聊天，这是产出文件。

镜头三：点“创建真实提醒”。  
展示它返回任务 ID。  
强调：不是“好的我会提醒你”，而是 Cron 里真的有任务。

镜头四：手机通过 Tailscale / PWA 打开同一个 OpenDeepSeek。  
强调：你可以躺在床上调度家里电脑上的 AI Agent。

### 2:10-2:45 飞轮逻辑

这件事背后其实是一个飞轮。

用户越多用真实任务，API 调用越多；  
API 不再只是聊天，而是进入 Agent 工作流；  
Agent 在真实任务里不断试错、修正、沉淀技能；  
用户也会越来越会指挥模型。

人、Agent、模型 API 三方形成闭环。  
这比单纯聊天有价值多了。

而且 DeepSeek V4 便宜到什么程度？  
它已经便宜到可以被大量用于日常自动化，而不是每次调用都心疼。

### 2:45-3:05 价值判断

我觉得国产模型不能只停留在“发布会很强”“榜单很高”。  
真正的支持，是把它接进真实工作流，让它每天帮我们做事。

不要只把 DeepSeek 当聊天软件。  
把它接到 Agent 上，让它读文件、写文件、跑任务、做网页、写周报、做 PPT。  

这才是我做 OpenDeepSeek 的原因。
