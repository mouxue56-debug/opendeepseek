# 国内网络问题

OpenDeepSeek CN 的目标是让中国用户不要卡死在 GitHub、Docker、Hugging Face、npm、pip 这些第一步依赖上。

## 1. 先跑网络体检

```bash
bash scripts/check-network-cn.sh
```

它会检查：

- DeepSeek API
- Gitee raw
- GitCode raw
- 阿里云 OSS manifest
- 腾讯云 COS manifest
- 国内容器镜像仓库
- 清华 PyPI 镜像
- npmmirror
- Gitee/GitCode Git 仓库
- 本地 OpenWebUI/Hermes 健康状态

如果只是本地服务没启动，看到 OpenWebUI/Hermes 不可达是正常的。

## 2. GitHub 打不开怎么办

国内版不要从 GitHub raw 单点开始。

推荐入口：

```bash
bash -c "$(curl -fsSL https://gitee.com/luoxueai/opendeepseek/raw/main/install-cn.sh)"
```

备用：

```bash
bash -c "$(curl -fsSL https://gitcode.com/mouxue56-debug/opendeepseek/raw/main/install-cn.sh)"
```

如果镜像仓库还没同步，可以手动指定：

```bash
OPDS_CN_GITEE_REPO=https://gitee.com/你的组织/opendeepseek.git bash install-cn.sh
```

## 3. Docker 镜像拉不下来怎么办

国内版优先走：

```text
registry.cn-hangzhou.aliyuncs.com/opendeepseek
```

如果你使用自己的 ACR/CCR 镜像仓库：

```bash
OPDS_IMAGE_REGISTRY=registry.cn-hangzhou.aliyuncs.com/your-namespace docker compose -f docker-compose.cn.yml up -d
```

如果仍然失败，使用离线镜像包：

```bash
OPDS_CN_OFFLINE=/path/to/opendeepseek-images-cn-amd64.tar.zst bash install-cn.sh
```

## 4. pip/npm 慢怎么办

`.env.example.cn` 默认设置：

```env
PIP_INDEX_URL=https://pypi.tuna.tsinghua.edu.cn/simple
UV_INDEX_URL=https://pypi.tuna.tsinghua.edu.cn/simple
NPM_CONFIG_REGISTRY=https://registry.npmmirror.com
PNPM_REGISTRY=https://registry.npmmirror.com
```

如果企业网络有内网源，可以改成公司自己的镜像。

## 5. Hugging Face 卡住怎么办

在线国内版不默认开启完全离线模式，因为没有预置模型时，RAG/文档解析可能会坏。

推荐路径：

1. 使用 OpenDeepSeek CN 镜像，里面预置中文 embedding/常用缓存。
2. 如果完全离线安装，使用离线包。
3. 离线包确认包含模型缓存后，再启用：

```env
OFFLINE_MODE=true
HF_HUB_OFFLINE=true
```

## 6. 搜索不可用怎么办

默认本地 SearXNG 只作为基础搜索后端。

后续可配置：

```env
OPDS_SEARCH_PROVIDER=auto
BOCHA_SEARCH_API_KEY=
JINA_API_KEY=
BING_SEARCH_V7_SUBSCRIPTION_KEY=
```

如果搜索不可用，OpenDeepSeek 应明确告诉用户“当前联网搜索不可用”，不能编造实时信息。

## 7. 当前发布状态提醒

`install-cn.sh`、`docker-compose.cn.yml` 和 `.env.example.cn` 是 OpenDeepSeek CN 的发布骨架。

真正对外宣称 China Ready 前，还需要：

- Gitee/GitCode 镜像完成同步。
- 国内镜像仓库发布对应镜像 tag。
- OSS/COS 离线包发布。
- `release-cn.json` 写入真实 checksum。
